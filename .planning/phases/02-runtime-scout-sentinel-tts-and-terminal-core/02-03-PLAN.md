# Plan 02-03: Sentinel Watcher and TTS Modules

<objective>
Implement the file watcher and Windows TTS modules as independent, smoke-testable core services.
</objective>

<context>
Read before executing:

- `.planning/phases/02-runtime-scout-sentinel-tts-and-terminal-core/02-RESEARCH.md`
- `AGENTS.md`
</context>

<tasks>
<task id="T1" type="auto">
  <title>Implement sentinel module</title>
  <files>
    <file>src/sentinel.js</file>
    <file>scripts/smoke-sentinel.js</file>
  </files>
  <action>
    Create `src/sentinel.js` using `chokidar`. Export `createSentinel(rootPath, options)` returning an object with `start()`, `stop()`, and event subscription support (`onEvent` callback or EventEmitter). Normalize events:
    - add -> `{ acao: "criado", arquivo, timestamp }`
    - change -> `{ acao: "modificado", arquivo, timestamp }`
    - unlink -> `{ acao: "removido", arquivo, timestamp }`

    Create `scripts/smoke-sentinel.js` that watches a temporary directory, creates/modifies/removes a file, waits for events, asserts all normalized actions appeared, stops the watcher, and exits. Use the `smoke:sentinel` script entry prepared by Plan 02-01.
  </action>
  <verify>
    Run `node --check src/sentinel.js`.
    Run `node --check scripts/smoke-sentinel.js`.
    Run `npm.cmd run smoke:sentinel`.
  </verify>
  <acceptance_criteria>
    - Watcher starts and stops cleanly.
    - Smoke test observes create/change/remove.
    - Emitted events have the required Portuguese shape.
  </acceptance_criteria>
</task>

<task id="T2" type="auto">
  <title>Implement TTS module</title>
  <files>
    <file>src/tts.js</file>
    <file>scripts/smoke-tts.js</file>
  </files>
  <action>
    Create `src/tts.js` that exports `speak(text, options)` and `buildSpeechCommand(text)`. Use PowerShell with `Add-Type -AssemblyName System.Speech` and `System.Speech.Synthesis.SpeechSynthesizer` on Windows. Return a structured result `{ ok, skipped, error }` and fail softly by default.

    Create `scripts/smoke-tts.js` that invokes `speak("SWARM inicializado", { timeoutMs: ... })`. It should not crash if TTS is unavailable; it should print the result and exit 0 for soft failure. Use the `smoke:tts` script entry prepared by Plan 02-01.
  </action>
  <verify>
    Run `node --check src/tts.js`.
    Run `node --check scripts/smoke-tts.js`.
    Run `npm.cmd run smoke:tts`.
  </verify>
  <acceptance_criteria>
    - TTS uses native PowerShell SpeechSynthesizer.
    - Smoke test exercises the TTS path.
    - TTS failure does not crash the app.
  </acceptance_criteria>
</task>
</tasks>
