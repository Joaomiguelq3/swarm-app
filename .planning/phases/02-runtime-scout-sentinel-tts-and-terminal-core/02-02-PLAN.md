# Plan 02-02: Project Scout Module

<objective>
Implement a reusable project scout that maps a folder, ignores generated/heavy directories, and returns both prompt-ready context and structured metadata.
</objective>

<context>
Read before executing:

- `.planning/phases/02-runtime-scout-sentinel-tts-and-terminal-core/02-CONTEXT.md`
- `.planning/phases/02-runtime-scout-sentinel-tts-and-terminal-core/02-RESEARCH.md`
- `AGENTS.md`
</context>

<tasks>
<task id="T1" type="auto">
  <title>Implement scout module</title>
  <files>
    <file>src/scout.js</file>
  </files>
  <action>
    Create `src/scout.js` with CommonJS exports:
    - `DEFAULT_IGNORE_DIRS`
    - `scoutProject(rootPath, options)`
    - `formatScoutContext(result)`

    `scoutProject` should recursively scan files under `rootPath`, skip ignored directories, cap file count and bytes/lines per file, and return:
    - `root`
    - `files`
    - `ignored`
    - `summary`
    - `context` as compact human-readable text

    Important files should include common project files such as `package.json`, `main.js`, `preload.js`, `AGENTS.md`, `CLAUDE.md`, README files, and files under `src/`/`renderer/`, while still respecting limits.
  </action>
  <verify>
    Run `node --check src/scout.js`.
  </verify>
  <acceptance_criteria>
    - API returns both string context and metadata.
    - Ignore list includes all required directories.
    - Output is bounded by options.
  </acceptance_criteria>
</task>

<task id="T2" type="auto">
  <title>Add scout smoke script</title>
  <files>
    <file>scripts/smoke-scout.js</file>

  <action>

  </action>
  <verify>

    Run `npm.cmd run smoke:scout`.

  <acceptance_criteria>

    - Output includes a compact summary and does not dump huge file contents.
  </acceptance_criteria>
</task>
</tasks>
