# Plan 01: Electron Foundation Scaffold

<objective>
Create the runnable Electron foundation for SWARM. The result must be a JavaScript-only desktop app scaffold where `npm start` opens `renderer/home.html`, Electron security boundaries are in place, and the required project files/directories exist for later phases.
</objective>

<context>
Read before executing:

- `.planning/PROJECT.md`
- `.planning/ROADMAP.md`
- `.planning/STATE.md`
- `.planning/phases/01-project-skeleton-and-electron-foundation/01-CONTEXT.md`
- `.planning/phases/01-project-skeleton-and-electron-foundation/01-RESEARCH.md`
- `AGENTS.md`
- `CLAUDE.md`
</context>

<scope>
## In Scope

- Initialize npm project metadata.
- Install/configure Electron with a working `npm start`.
- Create `.gitignore`, `.env`, and `brain.json`.
- Create `main.js`, `preload.js`, `renderer/home.html`, `renderer/workspace.html`, `renderer/style.css`, `renderer/app.js`, and `src/`.
- Implement a minimal preload IPC bridge and a renderer smoke check.
- Keep the first screen as a restrained SWARM home placeholder.

## Out of Scope

- Electron Forge or electron-builder.
- node-pty, xterm.js, chokidar, runtime spawning, scout, sentinel, TTS.
- Full Matrix rain, workspace cards, runtime badges, launch animation, or main workspace grid.
- Real API keys or token values.
</scope>

<tasks>
<task id="T1" type="auto">
  <title>Initialize package metadata and dependency setup</title>
  <files>
    <file>package.json</file>
    <file>package-lock.json</file>
  </files>
  <action>
    Create a JavaScript-only npm project for SWARM. Add `electron` as a development dependency using `npm.cmd` on Windows. Configure scripts:
    - `start`: launches Electron from the project root.
    - `check`: runs a lightweight Node syntax/config check suitable for CI-less local verification.

    Keep the package private. Do not add TypeScript, Electron Forge, electron-builder, React, Vite, or frontend frameworks in this phase.
  </action>
  <verify>
    Run `npm.cmd install` if dependencies are not installed yet.
    Run `npm.cmd run check`.
    Confirm `package.json` contains a working `start` script and no TypeScript/build framework dependencies.
  </verify>
  <acceptance_criteria>
    - `package.json` exists and is valid JSON.
    - `electron` is present in devDependencies.
    - `npm start` is configured.
    - No packaging framework is introduced.
  </acceptance_criteria>
</task>

<task id="T2" type="auto">
  <title>Create base project files and directories</title>
  <files>
    <file>.gitignore</file>
    <file>.env</file>
    <file>brain.json</file>
    <file>src/</file>
    <file>renderer/</file>
  </files>
  <action>
    Create the required base files and directories:
    - `.gitignore` excludes `node_modules/`, `.env.local`, logs, build outputs, and Electron/cache artifacts where appropriate.
    - `.env` contains only placeholder keys/comments, never real secrets.
    - `brain.json` contains a minimal valid JSON object for future session context.
    - `src/` exists, with an empty `.gitkeep` if needed so Git tracks the directory.
    - `renderer/` exists for UI files.

  </verify>
  <acceptance_criteria>

<task id="T3" type="auto">
  <title>Implement secure Electron main and preload boundary</title>
  <files>
    <file>main.js</file>
    <file>preload.js</file>
  </files>
  <action>
    Implement `main.js` so it creates a BrowserWindow and loads `renderer/home.html`. The BrowserWindow must use secure web preferences:
    - `contextIsolation: true`
    - `nodeIntegration: false`
    - `preload` points to `preload.js`

    Implement `preload.js` using `contextBridge.exposeInMainWorld` to expose a very small `swarm` API. Include a smoke method such as `getAppInfo()` or `ping()` backed by IPC from `main.js`. Do not expose raw `ipcRenderer`, `fs`, `child_process`, or broad Node access.
  </action>
  <verify>
    Run `node --check main.js`.
    Run `node --check preload.js`.
    Inspect `main.js` for `contextIsolation: true` and `nodeIntegration: false`.
  </verify>
  <acceptance_criteria>
    - Electron startup route is `renderer/home.html`.
    - `preload.js` exposes only a narrow bridge.
    - Renderer cannot access Node directly.
  </acceptance_criteria>
</task>

<task id="T4" type="auto">
  <title>Create minimal renderer placeholders</title>
  <files>
    <file>renderer/home.html</file>
    <file>renderer/workspace.html</file>
    <file>renderer/style.css</file>
    <file>renderer/app.js</file>
  </files>
  <action>
    Create a restrained terminal-style placeholder UI:
    - `home.html` shows SWARM as the first screen and indicates the Electron shell is running.
    - `workspace.html` exists as a placeholder for later phases but is not loaded by default.
    - `style.css` defines the core dark terminal palette from the PRD without implementing full Phase 4 visual polish.
    - `app.js` calls the safe preload API and renders a small runtime/app status indicator.

    Do not implement workspace cards, Matrix rain, terminal panes, runtime dropdowns, or launch animations in this phase.
  </action>
  <verify>
    Run `node --check renderer/app.js`.
    Confirm `home.html` references `style.css` and `app.js`.
    Confirm `workspace.html` exists.
  </verify>
  <acceptance_criteria>
    - `home.html` is the startup UI.
    - `workspace.html` is present as placeholder.
    - UI is simple, dark, and does not claim later features are implemented.
  </acceptance_criteria>
</task>
