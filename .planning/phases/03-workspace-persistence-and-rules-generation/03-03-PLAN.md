# Plan 03-03: Phase 3 Aggregate Verification

<objective>
Run an aggregate Phase 3 verification that proves workspace persistence, runtime/model storage, `brain.json` bootstrap, runtime rules generation, and safe IPC are all ready for Phase 4 UI integration.
</objective>

<context>
Read before executing:

- `.planning/phases/03-workspace-persistence-and-rules-generation/03-CONTEXT.md`

<scope>
## In Scope

- Aggregate smoke command.
- Phase 3 summary.
- Verification of all Phase 3 success criteria.

## Out of Scope

- Polished Home UI.
- Main workspace cockpit UI.
- Runtime process restart behavior.
- End-to-end Claude/Codex agent demos.
</scope>

<threat_model>
Potential issues:

- Passing isolated module tests while aggregate project checks fail.
- Smoke tests accidentally writing into real user app data or real project roots.
- Phase 3 summary overstating UI or process lifecycle completion.

Mitigations:

- Aggregate smoke runs `smoke:workspaces`, `smoke:workspace-ipc`, and `check`.
- Smoke scripts must use temporary directories.
- Summary explicitly records deferred UI/runtime lifecycle scope.
</threat_model>
