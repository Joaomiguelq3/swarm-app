# Plan 03-02: Workspace IPC Bridge

<objective>
Expose the Phase 3 workspace persistence layer through a safe Electron IPC boundary so later UI phases can create, list, update, and touch saved workspaces without renderer Node access.
</objective>

<context>
Read before executing:

- `.planning/phases/03-workspace-persistence-and-rules-generation/03-CONTEXT.md`
- `.planning/phases/03-workspace-persistence-and-rules-generation/03-RESEARCH.md`
- `.planning/phases/03-workspace-persistence-and-rules-generation/03-01-SUMMARY.md` if present
- `src/workspaces.js`
- `main.js`
- `preload.js`
- `AGENTS.md`
</context>

<scope>
## In Scope

- Minimal workspace IPC registration helper.
- Main-process handler registration.
- Safe preload workspace API.
- Smoke test for IPC helper and preload API shape.

## Out of Scope

- Full Home workspace selector cards.
- Runtime switching UI.
- Real folder dialog UX.
- Agent spawning or terminal panes.
</scope>

<threat_model>
Potential issues:

- Renderer could gain broad IPC or filesystem access.
- IPC handlers could accept malformed payloads and write to unintended locations.
- Main process could become cluttered with business logic.

Mitigations:

- Expose explicit preload methods only.
- Validate payloads before calling `src/workspaces.js`.
- Register handlers in a small `src/workspace-ipc.js` helper and keep persistence in `src/workspaces.js`.
</threat_model>

<tasks>
<task id="T1" type="auto">
  <title>Create workspace IPC registration helper</title>
  <files>
    <file>src/workspace-ipc.js</file>
  </files>
  <action>
    Create `src/workspace-ipc.js` exporting `registerWorkspaceIpc({ ipcMain, app })`. Register handlers with explicit channel names:
    - `swarm:workspaces:list`
    - `swarm:workspaces:create`
    - `swarm:workspaces:update-runtime`
    - `swarm:workspaces:update-model`
    - `swarm:workspaces:touch`

    Each handler should call the corresponding `src/workspaces.js` function and return structured results. Do not expose arbitrary file operations or a generic "invoke any channel" helper.
  </action>
  <verify>
    Run `node --check src/workspace-ipc.js`.
  </verify>
  <acceptance_criteria>
    - IPC helper registers all required workspace channels.
    - Handlers accept explicit, narrow payloads.
    - Business logic remains delegated to `src/workspaces.js`.
  </acceptance_criteria>
</task>

<task id="T2" type="auto">
  <title>Wire main and preload APIs</title>
  <files>
    <file>main.js</file>
    <file>preload.js</file>
  </files>
  <action>
    Update `main.js` to import and call `registerWorkspaceIpc({ ipcMain, app })` once during startup setup, preserving the existing `swarm:get-app-info` handler and Electron security settings.

    Update `preload.js` to expose:
    - `window.swarm.workspaces.list()`
    - `window.swarm.workspaces.create(input)`
    - `window.swarm.workspaces.updateRuntime(id, runtime, model)`
    - `window.swarm.workspaces.updateModel(id, model)`
    - `window.swarm.workspaces.touch(id)`

    Preserve `window.swarm.getAppInfo()`. Do not expose raw `ipcRenderer`, `fs`, or broad Node APIs.
  </action>
  <verify>
    Run `node --check main.js`.
    Run `node --check preload.js`.
  </verify>
  <acceptance_criteria>
    - Main registers workspace IPC without moving persistence logic into `main.js`.
    - Preload exposes narrow workspace methods.
    - Existing app info smoke API still exists.
  </acceptance_criteria>
</task>

</tasks>
