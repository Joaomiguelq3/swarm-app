# Plan 05-03: Workspace UI Smoke and Phase Verification

<objective>
Add static smoke coverage for the main workspace UI and run aggregate Phase 5 verification so the cockpit, runtime/model controls, stop hook, pane indicators, and deferred orchestration boundaries are checked before execution is marked complete.
</objective>

<context>
Read before executing:

- `.planning/phases/05-main-workspace-ui-and-runtime-switching/05-CONTEXT.md`
- `.planning/phases/05-main-workspace-ui-and-runtime-switching/05-RESEARCH.md`
- `.planning/phases/05-main-workspace-ui-and-runtime-switching/05-01-SUMMARY.md` if present
- `.planning/phases/05-main-workspace-ui-and-runtime-switching/05-02-SUMMARY.md` if present
- `renderer/home.html`
- `renderer/workspace.html`
- `renderer/style.css`
- `renderer/app.js`
- `package.json`
- `scripts/smoke-home-ui.js`

- `AGENTS.md`
</context>

<scope>
## In Scope

- Static smoke test for workspace route DOM hooks, CSS classes, and renderer behavior markers.
- Aggregate `smoke:phase5` command.
- `package.json` check script coverage for new scripts.
- Phase 5 summary with verification results, requirement coverage, and deferred scope.

## Out of Scope

- Full Playwright/browser automation unless already available.
- Screenshot-baseline visual regression.
- Real Claude/Codex process lifecycle tests.
- Real file watcher or TTS tests beyond existing scripts.
</scope>

<threat_model>
Potential issues:

- Static tests can miss visual overlap.
- Smoke tests may assert brittle implementation details.
- Verification may overstate Phase 5 by implying real agents work.

Mitigations:

- Check stable ids/classes/API references required by the phase contract rather than incidental formatting.
- Run `npm.cmd run check` and `npm.cmd run smoke:phase5`.
- Summary must separate delivered UI/lifecycle hooks from deferred real orchestration.
</threat_model>
