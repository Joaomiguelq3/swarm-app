<objective>
Replace the Phase 5 launch placeholder with real renderer orchestration behavior: mission launch payloads, event subscriptions, pane output/status updates, file/lifecycle feed entries, and stop hook connection.

<scope>
## In Scope

- Mission launch payload from active workspace, mission input, agent count, runtime, and model.
- Event subscription lifecycle on workspace page.
- Pane status updates from orchestration events.
- Terminal output rendering inside pane surfaces.
- Unified feed entries for mission, agent, file, stop, and error events.
- Runtime switch stop hook connected to orchestration stop.
- Stop/launch UI state while a mission is active.

## Out of Scope

- Launch overlay animation.
- TTS implementation internals.
- Real xterm.js deep terminal emulation if DOM output is enough for Phase 6 smoke.
- Real Claude/Codex demo verification.
</scope>

<threat_model>
Potential issues:

- Renderer subscribes multiple times and duplicates events.
- Long output causes layout shifts or feed spam.
- Invalid mission/agent count launches bad core input.
- Runtime switch leaves active processes running.

Mitigations:

- Store unsubscribe cleanup and call it when reinitializing.
- Append raw output to bounded pane output areas and summarize feed entries.
- Validate mission non-empty and clamp agent count before launch.
- Await orchestration stop from `stopActiveProcesses()`.
</threat_model>
