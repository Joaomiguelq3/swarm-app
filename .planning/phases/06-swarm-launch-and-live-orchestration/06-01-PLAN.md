# Plan 06-01: Swarm Orchestrator Core and Fake Runtime

<objective>
Create the runtime-agnostic orchestration core in `src/swarm.js`: deterministic mission decomposition, scout context inclusion, parallel process launch through `node-pty`, lifecycle events, stop support, and fake-runtime smoke coverage.
</objective>

<context>
Read before executing:

- `.planning/phases/06-swarm-launch-and-live-orchestration/06-CONTEXT.md`
- `.planning/phases/06-swarm-launch-and-live-orchestration/06-RESEARCH.md`
- `.planning/phases/05-main-workspace-ui-and-runtime-switching/05-VERIFICATION.md`
- `.planning/phases/02-runtime-scout-sentinel-tts-and-terminal-core/02-VERIFICATION.md`
- `src/runtimes.js`
- `src/scout.js`
- `scripts/smoke-pty.js`
- `package.json`
- `AGENTS.md`
</context>

<scope>
## In Scope

- `src/swarm.js`.
- Deterministic task decomposition.
- Scout context integration through `src/scout.js`.
- Runtime lookup through `src/runtimes.js`.
- `node-pty` process spawning for N agents.
- Event emitter contract for mission, agent, output, status, exit, and error events.
- Stop method that terminates active child processes.
- Fake runtime/spawn injection for deterministic smoke verification.

## Out of Scope

- Main/preload IPC.
- Renderer event handling.
- Sentinel file event IPC.
- Launch overlay.
- Real Claude/Codex demo execution.
</scope>

<threat_model>
Potential issues:

- Real runtime CLIs may not be installed.
- Agent prompts may omit workspace context.
- Child processes may leak if stop is not explicit.
- Tests may become nondeterministic if they spawn real runtimes.

Mitigations:

- Add injectable runtime/spawn options for smoke tests.
- Include scout context and task metadata in every prompt.
- Track spawned processes in the core and expose a stop method.
- Smoke-test with a fake runtime only.
</threat_model>
