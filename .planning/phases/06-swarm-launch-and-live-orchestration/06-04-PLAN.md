# Plan 06-04: Launch Overlay and Lifecycle Polish

<objective>
Add the Phase 6 launch overlay and final lifecycle polish so mission launch visibly transitions into live pane activity and uses best-effort TTS lifecycle behavior without blocking orchestration.
</objective>

<context>
Read before executing:

- `.planning/phases/06-swarm-launch-and-live-orchestration/06-CONTEXT.md`
- `.planning/phases/06-swarm-launch-and-live-orchestration/06-RESEARCH.md`
- `.planning/phases/06-swarm-launch-and-live-orchestration/06-03-SUMMARY.md` if present
- `renderer/workspace.html`
- `renderer/style.css`
- `renderer/app.js`
- `src/swarm-ipc.js`
- `src/tts.js`
- `AGENTS.md`
</context>

<scope>
## In Scope

- Launch overlay DOM.
- Overlay CSS and animation states.
- Runtime badge in overlay.
- Progressive task list reveal.
- Pane glow during launch.
- Fade-out when launch starts/agents are spawned.
- Smoke coverage for overlay hooks and lifecycle markers.

## Out of Scope

- Pixel-perfect animation testing.
- Generated images/assets.
- Real demo rehearsal.
- App-close cleanup hardening.
</scope>

<threat_model>
Potential issues:

- Overlay blocks the UI after a launch error.
- Overlay hides errors or misrepresents fake/demo state.
- Animation causes layout shifts.

Mitigations:

- Always clear overlay on mission start/error/stop timeout paths.
- Feed/panes remain authoritative for errors.
- Use fixed overlay layer and stable task rows.
</threat_model>
