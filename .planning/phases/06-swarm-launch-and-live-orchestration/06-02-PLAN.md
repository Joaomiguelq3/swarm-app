# Plan 06-02: Swarm IPC, Sentinel, and TTS Integration

<objective>
Connect the `src/swarm.js` core to Electron through main/preload IPC, main-process active swarm registry, sentinel file events, and best-effort TTS lifecycle announcements.
</objective>

<context>
Read before executing:

- `.planning/phases/06-swarm-launch-and-live-orchestration/06-CONTEXT.md`
- `.planning/phases/06-swarm-launch-and-live-orchestration/06-RESEARCH.md`
- `.planning/phases/06-swarm-launch-and-live-orchestration/06-01-SUMMARY.md` if present

- `src/tts.js`

- `main.js`
- `preload.js`
- `package.json`

</context>

<scope>
## In Scope

- New `src/swarm-ipc.js` registration helper.
- Main-process active swarm registry.
- IPC invoke channels for launch and stop.
- IPC event channels for mission/agent/output/file/TTS events.
- Preload orchestration API with safe event subscription/unsubscription.
- Sentinel start/stop around active missions.
- Best-effort TTS calls for agent completion/errors and mission completion.
- Smoke coverage with fake runtime/core injection.

## Out of Scope

- Renderer UI wiring.
- Launch overlay DOM/CSS.
- App-close final hardening beyond current mission stop.
- Real Claude/Codex demo verification.
</scope>

<threat_model>
Potential issues:

- Raw `ipcRenderer` leaks into renderer.
- Multiple missions run at once and conflict.
- Sentinel watcher leaks after mission stop.
- TTS failure breaks mission.

Mitigations:

- Expose only narrow preload methods and subscription cleanup callbacks.
- Reject or stop existing active mission before a new launch.
- Always stop sentinel when mission stops/completes/errors.
- Wrap TTS in best-effort calls and emit warnings only.
</threat_model>
