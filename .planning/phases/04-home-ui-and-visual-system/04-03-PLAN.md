# Plan 04-03: Home UI Smoke and Verification

<objective>
Add lightweight verification for the Home UI and run the aggregate Phase 4 checks so the phase can be validated in both GUI and non-GUI environments.
</objective>

<context>
Read before executing:

- `.planning/phases/04-home-ui-and-visual-system/04-CONTEXT.md`
- `.planning/phases/04-home-ui-and-visual-system/04-RESEARCH.md`
- `.planning/phases/04-home-ui-and-visual-system/04-01-SUMMARY.md` if present
- `.planning/phases/04-home-ui-and-visual-system/04-02-SUMMARY.md` if present
- `renderer/home.html`
- `renderer/style.css`
- `renderer/app.js`
- `package.json`
- `AGENTS.md`
</context>

<scope>
## In Scope

- Static smoke test for Home UI hooks and integration references.
- Aggregate `smoke:phase4` command.
- Phase 4 summary with verification results and deferred scope.

## Out of Scope

- Full browser automation framework unless already available.
- Screenshot-baseline visual regression.
- Terminal panes or agent lifecycle tests.
</scope>

<threat_model>
Potential issues:

- Static smoke tests can miss visual overlap.
- GUI launch may be blocked in the execution environment.
- Verification might overstate what Phase 4 delivered.

Mitigations:

- Smoke tests check concrete DOM/CSS/JS hooks, then manual GUI launch is attempted when possible.
- If GUI launch is blocked, document the limitation.
- Summary explicitly lists deferred Phase 5/6 scope.
</threat_model>
