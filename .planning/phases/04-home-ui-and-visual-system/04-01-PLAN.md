# Plan 04-01: Home Shell and Visual System

<objective>
Replace the placeholder Home route with the static structure and visual system needed for the SWARM workspace selector: compact console layout, typing logo hook, Matrix canvas hook, runtime badge classes, buttons, form controls, and responsive foundations.
</objective>

<context>
Read before executing:

- `.planning/PROJECT.md`
- `.planning/REQUIREMENTS.md`
- `.planning/ROADMAP.md`
- `.planning/phases/04-home-ui-and-visual-system/04-CONTEXT.md`
- `.planning/phases/04-home-ui-and-visual-system/04-RESEARCH.md`
- `renderer/home.html`
- `renderer/style.css`
- `renderer/app.js`
- `preload.js`
- `AGENTS.md`
</context>

<scope>
## In Scope

- Home screen DOM structure for logo, Matrix canvas, toolbar, workspace grid, state regions, and new workspace modal shell.
- CSS variables and reusable UI classes for terminal theme, badges, cards, buttons, inputs, modal, and responsive grid.
- Typing logo and Matrix canvas initialization hooks in `renderer/app.js`.

## Out of Scope

- Real terminal panes.
- Main workspace cockpit.
- Runtime process lifecycle.
- Agent spawning.
- Launch overlay.
</scope>

<threat_model>
Potential issues:

- Renderer accidentally gains Node access.
- Animation hurts readability or performance.
- Layout becomes a landing page instead of an operational tool.

Mitigations:

- Keep renderer code browser-only and use only `window.swarm` preload methods.
- Use low-opacity canvas and modest animation pacing.
- Keep the logo compact and workspace grid visible.
</threat_model>
